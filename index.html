Multitarea con FreeRTOS

Dos tareas corren en paralelo sobre el ESP32. tareaSensor lee el DHT22 cada 2 s. tareaWeb atiende peticiones HTTP. El planificador de FreeRTOS distribuye el CPU entre ambas.

🔒

Mutex para exclusión mutua

La variable temperatura es un recurso compartido. Un SemaphoreHandle_t garantiza que solo una tarea acceda a ella por vez, evitando condiciones de carrera.

🌐

Servidor web embebido

El ESP32 expone un servidor HTTP en el puerto 80 a través de su WiFi integrado. La página se auto-refresca cada 2 s mostrando la temperatura en tiempo real desde el navegador.

🌡

Sensor DHT22

Conectado al GPIO 15. Mide temperatura con resolución de 0.1 °C. La lectura usa dht.readTemperature() y se valida con isnan() para descartar valores inválidos.

// 04 — Flujo de tareas FreeRTOS

tareaSensor()

Prioridad 2
01Espera vTaskDelay(2000ms) cediendo el CPU
02Lee dht.readTemperature()
03Valida con isnan() — descarta si falla
04Toma el MUTEX con xSemaphoreTake()
05Escribe temperatura = temp
06Libera MUTEX con xSemaphoreGive()
MUTEX

tareaWeb()

Prioridad 1
01Llama server.handleClient() continuamente
02Al recibir GET / invoca handleRoot()
03Toma el MUTEX con xSemaphoreTake()
04Lee tempActual = temperatura
05Libera MUTEX con xSemaphoreGive()
06Envía página HTML con server.send(200)
⚿

SemaphoreHandle_t mutex

Protege la variable float temperatura — recurso compartido entre ambas tareas. Solo una tarea puede tenerlo a la vez. Si está tomado, la otra tarea queda bloqueada hasta que se libere.
